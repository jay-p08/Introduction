def judge_note(note_time, input_time):
    diff = abs(note_time - input_time)
    if   diff < 40:  return "PERFECT"
    elif diff < 80:  return "GOOD"
    elif diff < 120: return "BAD"

핵심 코드 — AES 파일 암호화
from Crypto.Cipher import AES
import os

def encrypt_file(path, key):
    iv = os.urandom(16)
    cipher = AES.new(key, AES.MODE_CBC, iv)
    with open(path, 'rb') as f:
        data = f.read()
    encrypted = cipher.encrypt(pad(data, 16))
    return iv + encrypted
▸ 설계 의도

파일마다 다른 IV를 생성해 동일한 키로 암호화해도 암호문이 달라지도록 설계했습니다. IV는 암호문 앞에 붙여 복호화 시 분리합니다.

▸ 배운 점

CBC 모드에서 패딩이 없으면 블록 크기 불일치로 오류가 발생합니다. 암호화는 알고리즘보다 키/IV 관리 방식이 보안의 핵심임을 배웠습니다.

핵심 코드 — Blinn-Phong 픽셀 셰이더 (HLSL)
float3 BlinnPhong(float3 N, float3 L,
                  float3 V, float3 diff,
                  float  shininess) {
    float3 H     = normalize(L + V);
    float  NdotL = max(dot(N, L), 0.0);
    float  NdotH = max(dot(N, H), 0.0);
    float3 spec  = pow(NdotH, shininess);
    return diff * NdotL + spec;
}
▸ 설계 의도

Phong 대비 하프벡터(H)를 사용해 계산 비용을 줄이면서 비슷한 시각 품질을 냅니다. shininess 값이 클수록 하이라이트가 날카로워집니다.

▸ 배운 점

max(dot, 0)으로 음수를 막지 않으면 빛이 반대쪽에서 새어나오는 버그가 발생함을 직접 확인했습니다.

boolean checkAABB(Rect a, Rect b) {
    return a.x < b.x + b.w
        && a.x + a.w > b.x
        && a.y < b.y + b.h
        && a.y + a.h > b.y;
}
void resolveCollision(Entity e, Tile t) {
    float overlapX = Math.min(
        e.right() - t.left(), t.right() - e.left());
    e.x += (e.centerX() < t.centerX())
        ? -overlapX : overlapX;
}
▸ 설계 의도

축 분리 이론(SAT)의 2D 간소화 버전입니다. 겹침 거리(overlap)를 측정해 가장 짧은 축으로 밀어내 물체가 벽을 통과하지 않도록 합니다.

▸ 배운 점

속도가 빠른 오브젝트는 한 프레임에 벽을 통과할 수 있습니다(터널링). Continuous Collision Detection이 필요하다는 것을 이 프로젝트에서 처음 인식했습니다.

핵심 코드 — 블록 회전 행렬
int[][] rotate(int[][] shape) {
    int n = shape.length;
    int[][] rotated = new int[n][n];
    for (int r = 0; r < n; r++)
        for (int c = 0; c < n; c++)
            rotated[c][n-1-r] = shape[r][c];
    return rotated;
}
▸ 설계 의도

2D 배열 전치(transpose) + 열 반전으로 시계 방향 90도 회전을 구현합니다. 별도의 회전 상태 배열 없이 수식 하나로 처리됩니다.

▸ 배운 점

선형 대수의 행렬 변환이 실제 게임 로직에 그대로 쓰인다는 것을 처음 체감했습니다. 회전 후 벽 충돌 체크를 반드시 병행해야 자연스러운 SRS 회전이 됩니다.

void AMyWeapon::Fire() {
  FVector Start = Camera->GetComponentLocation();
  FVector End   = Start +
    Camera->GetForwardVector() * Range;

    Hit, Start, End, ECC_Visibility);
